# 项目路线图 (Roadmap)

## 阶段一：基础设施搭建 (已完成)

- [x] 初始化项目结构 (uv, src layout)。
- [x] 配置 CI/CD (GitHub Actions)。
- [x] 升级 Python 环境至 3.14。
- [x] **架构转型**: 从 API 服务转型为离线数据处理架构。
- [x] **模型定义**: 使用 Pydantic 定义 `Stream` 和 `Dialogue` 数据结构。

## 阶段二：数据积累与清洗 (进行中)

- [ ] **爬虫对接**:
  - [ ] 爬虫端生成符合 `Stream` 和 `Dialogue` 模型定义的 JSON 数据。
  - [ ] 配置定时任务 (Cron/Task Scheduler) 将数据文件同步至本项目目录。
- [ ] **数据校验脚本**:
  - [ ] 编写脚本批量读取 JSON 文件。
  - [ ] 使用 Pydantic 模型校验数据完整性和类型。
- [ ] **数据清洗**:
  - [ ] 去除无效对话（如仅包含表情包、长度过短）。
  - [ ] 格式化时间戳。

## 阶段三：AI 模型微调 (Fine-tuning)

- [ ] **数据集构建**: 将清洗后的数据转换为模型训练格式 (如 JSONL, ShareGPT 格式)。
- [ ] **模型选择**: 评估适合的基座模型 (如 LLaMA, Qwen 等)。
- [ ] **训练流程**:
  - [ ] 搭建训练脚本 (LoRA/QLoRA)。
  - [ ] 在本地或云端进行微调实验。
- [ ] **模型评估**: 测试微调后模型的对话风格是否接近目标 VTB。

## 阶段四：应用集成

- [ ] **推理服务**: 将微调后的模型封装为 API。
- [ ] **直播集成**: 实现自动回复弹幕功能。
- [ ] **语音合成 (TTS)**: 集成 VTB 声音克隆。
